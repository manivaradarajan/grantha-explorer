### **THE GOLDEN RULE**

**YOU MUST NOT CHANGE, ADD, OR DELETE A SINGLE CHARACTER OF THE DEVANAGARI TEXT FOR ANY REASON.** The final Devanagari content in the output must be a perfect, 1-to-1 copy of the Devanagari content from the input, in the exact same order.

---
### **Prohibited Actions**

You are explicitly forbidden from performing these actions:
*   **You ARE NOT an editor.** Do not "fix" what you perceive as typos or errors.
*   **You DO NOT remove metadata.** Text like `०३०१श्रीः`, `[some text]`, or `प्रकाशिकारङ्गरामानुजमुनिविरචिता` is content. You MUST include it.
*   **You DO NOT change the order of the content.** The text must appear in the same sequence it was given in the input.

### **Your Workflow**

Follow this precise process:
1.  Create the YAML frontmatter block based on the input file's content.
2.  After the YAML block, begin copying the input text verbatim.
3.  For **Mantra** sections, insert the appropriate Markdown header (`# Mantra X.Y.Z`).
4.  For **Commentary** sections, you must perform two steps in order:
    a. **FIRST, insert a machine-readable metadata comment.** The format is exactly: `<!-- commentary: {"commentary_id": "...", "passage_ref": "..."} -->`.
       *   Fill the `commentary_id` with the commentary's unique ID from the YAML frontmatter (e.g., `ranga-ramanujamuni-prakashika`).
       *   Fill the `passage_ref` with the reference of the mantra that the commentary is for (e.g., `3.1.1`).
    b. **SECOND, on the next line,** insert the standard Markdown header (`# Commentary: X.Y.Z`).
5.  Enclose all Sanskrit text blocks in the required `<!-- sanskrit:devanagari -->` tags.
6.  Continue this process until all input text has been transferred to the output.
7.  Preserve paragraph breaks.
8.  Preserve bold markup ("**").
